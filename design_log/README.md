# Design Log

이 폴더는 GS-Assisted Triangle Splatting+ 실험의 **설계 변천사**를 시간순으로 기록한다.

## 역할 분담

- **`../EXPERIMENT.md`** = 항상 **현재 설계 스냅샷**만 담는다. "지금 코드가 무엇을 하는가"를
  읽는 곳. 과거 설계·diff·진단 서술은 넣지 않는다.
- **`design_log/`** (여기) = **바뀐 이유와 변경점**을 날짜별로 쌓는다. "왜 지금 설계가
  이렇게 됐나 / 무엇을 어떻게 바꿨나"를 읽는 곳.

## 규칙

- 설계를 바꿀 때마다 새 엔트리 파일을 추가한다: `YYYY-MM-DD-<slug>.md`.
- 엔트리는 **트리거(무슨 결과/문제) → 진단 → 결정 → 코드 변경점(파일별) → 검증** 순으로.
- 엔트리는 **불변 기록**이다. 나중에 틀린 걸로 밝혀져도 지우지 말고, 다음 엔트리에서
  바로잡는다(감사 추적).
- 엔트리를 추가하면 아래 목록에 한 줄 추가하고, `EXPERIMENT.md`를 새 현재상태로 갱신한다.
